## Lección: Proyectos Prácticos Intermedios

### 1. Proyecto 1: Gestor de Tareas

#### Objetivos
- Diseñar y desarrollar un gestor de tareas utilizando Python.
- Implementar funcionalidades básicas como agregar, eliminar y listar tareas.
- Utilizar estructuras de datos adecuadas para almacenar y manipular la información de las tareas.

#### Descripción del Proyecto
El gestor de tareas es una aplicación que permite a los usuarios administrar sus tareas diarias. El proyecto se centrará en la implementación de las siguientes funcionalidades:

##### Funcionalidades Principales
1. **Agregar Tarea:** Permitirá al usuario agregar una nueva tarea especificando su descripción y fecha de vencimiento opcionalmente.
2. **Eliminar Tarea:** Permite al usuario eliminar una tarea existente basada en su identificador.
3. **Listar Tareas:** Muestra todas las tareas almacenadas, junto con sus detalles como descripción, estado (pendiente/completada), y fecha de vencimiento si está definida.

##### Implementación
- Utilizarás estructuras de datos como listas o diccionarios para almacenar las tareas.
- Implementarás funciones y métodos para manejar la lógica de negocio del gestor de tareas.
- Podrías considerar persistir los datos utilizando archivos o incluso una base de datos simple (opcional).

### 2. Proyecto 2: API para un Blog Simple

#### Objetivos
- Diseñar y construir una API RESTful básica para gestionar un blog simple utilizando Python y Flask.
- Implementar operaciones CRUD (Crear, Leer, Actualizar, Eliminar) para manipular entradas de blog.
- Utilizar `requests` para probar la API y asegurar su funcionalidad.

#### Descripción del Proyecto
El API para un blog simple permitirá a los usuarios realizar las siguientes operaciones:

##### Operaciones CRUD
1. **Crear Entrada:** Permite a los usuarios crear nuevas entradas de blog especificando título, contenido y fecha de publicación.
2. **Leer Entrada:** Muestra detalles de una entrada de blog basada en su identificador único.
3. **Actualizar Entrada:** Permite actualizar el título o el contenido de una entrada existente.
4. **Eliminar Entrada:** Permite eliminar una entrada de blog existente basada en su identificador.

##### Implementación
- Utilizarás el framework Flask para construir la API.
- Definirás rutas (`routes`) y métodos HTTP adecuados (`GET`, `POST`, `PUT`, `DELETE`) para cada operación CRUD.
- Implementarás la lógica para interactuar con una estructura de datos (como listas o diccionarios) para almacenar temporalmente las entradas del blog.
- Utilizarás `requests` para probar la API y verificar su funcionamiento.

### Resumen y Tareas

- En esta lección, exploraste dos proyectos prácticos intermedios que te permitirán aplicar tus conocimientos de Python para desarrollar aplicaciones útiles y funcionales.
- El primer proyecto se enfoca en construir un gestor de tareas simple, mientras que el segundo proyecto te desafía a crear una API RESTful para un blog.
- Practica implementando estas soluciones y asegúrate de comprender los principios de diseño y desarrollo detrás de cada proyecto.
- **Tarea:** Elige uno de los proyectos y comienza a implementarlo. Documenta tu proceso, desafíos enfrentados y soluciones encontradas.
